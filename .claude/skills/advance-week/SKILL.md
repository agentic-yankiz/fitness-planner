# Advance Week

Move Shaked forward in the 4-week-wave-plus-deload cycle, using his logged numbers.

## Steps
1. Read `knowledge/training-principles.md` (progression rules) and the latest
   `tracking/week-*.md` logs.
2. Identify the current position in the cycle (Week 1–4, or deload).
3. Apply the rules:
   - **Loading week → next loading week:** advance the levers per the "How each week gets
     harder" table in `PLAN.md` — but only on exercises that were hit **at/under target
     RPE** and **without rising shoulder pain**. Hold or regress anything that wasn't.
   - **Week 4 → deload (Week 5):** cut volume ~half, drop intensity to RPE 5–6, keep
     technique and prehab. Prompt for benchmark re-tests.
   - **Deload → new block:** set **new baselines** from the best clean Week 3–4 numbers and
     update them in `PLAN.md` and `athlete-profile.md`. Consider progressing a skill level
     (e.g., planche lean → tuck planche; archer → assisted one-arm) if criteria are met.
4. **Shoulder gate:** never advance planche-lean depth or one-arm lock-off load if
   front-shoulder pain is trending up — defer to the `physio-advisor` mindset and hold.

## Output rules
- State the new week/block and the **per-exercise decision (advance / hold / regress)** with
  the reason from the logs.
- Apply the changes to `PLAN.md` (and baselines in `athlete-profile.md` on a new block),
  keeping `PLAN.md` human-readable.
- Summarize in a few lines what changed and why.
