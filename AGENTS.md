# Fitness Planner Agent Entry Point

Read `CLAUDE.md` first. It defines how to work in this repository.

Before project work, use the org Kanban/MCP baseline:

- Check `agentic-yankiz/fitness-planner` on `Fitness Planner Kanban`.
- Prefer the GitHub MCP/app connector for issue, PR, label, and Project updates when available.
- If no connector is available, use `gh` authenticated as `yankihermesapp[bot]`.
- Keep `type:*`, `priority:*`, `area:*`, and `status:*` labels current.
- Move the Project card on every transition; never leave completed work in `No Status`.
